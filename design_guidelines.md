# 拖延症消除神器 - 设计指南

## 品牌定位

**产品定位**：一个帮助用户克服拖延、高效完成目标的智能任务管理工具。

**设计风格**：清晰明亮的健身房风格 - 像健身计划一样拆解任务，每完成一步都有达成小目标的爽快感。

**目标用户**：有目标但容易拖延的年轻人、学生、职场人士。

## 配色方案

### 主色调
- **活力橙** `bg-orange-500` / `text-orange-500` - 代表能量、成就感、进度完成
- **清新蓝** `bg-blue-500` / `text-blue-500` - 代表思考、规划、冷静分析

### 中性色
- **背景色** `bg-white` - 干净明亮的背景
- **次要背景** `bg-gray-50` - 卡片和分隔区域
- **文本色** `text-gray-900` - 主要文字
- **次要文本** `text-gray-500` - 辅助说明文字
- **边框色** `border-gray-200` - 细腻的分割线

### 语义色
- **成功** `bg-green-500` - 任务完成
- **警告** `bg-yellow-500` - 即将到期的任务
- **错误** `bg-red-500` - 已过期的DDL

## 字体规范

### 字体家族
- **中文**：Noto Sans SC
- **英文/数字**：Inter

### 排版层级
- **H1 - 页面标题**：`text-2xl font-bold text-gray-900`
- **H2 - 卡片标题**：`text-lg font-semibold text-gray-900`
- **H3 - 步骤标题**：`text-base font-medium text-gray-900`
- **Body - 正文**：`text-sm text-gray-700`
- **Caption - 辅助文字**：`text-xs text-gray-500`

## 组件使用原则

### 通用组件优先级
所有通用 UI 组件必须优先从 `@/components/ui/*` 导入：

**必须优先使用的组件**：
- `Button` - 所有按钮（主操作、次级操作、取消操作）
- `Input` / `Textarea` - 任务输入、备注输入
- `Card` - 任务卡片容器
- `Checkbox` - 任务步骤复选框
- `Progress` - 进度条展示
- `Tabs` - 模式切换（基础模式/干大事模式）
- `Switch` - 开关选项
- `Badge` - 状态标签（进行中/已完成/已过期）
- `Dialog` / `AlertDialog` - 确认弹窗
- `Toast` / `Sonner` - 操作反馈提示
- `Skeleton` - 加载状态

**禁止手搓通用组件**：
- 禁止用 `View/Text` + Tailwind 手搓按钮、输入框、弹窗、Tabs、卡片等通用 UI
- 组件库缺失时，优先补齐到 `src/components/ui`，不要在页面内临时造轮子

### 容器样式
- **卡片**：`bg-white rounded-2xl border border-gray-200 shadow-sm`
- **圆角**：统一使用 `rounded-2xl`（大卡片）、`rounded-xl`（小元素）
- **阴影**：`shadow-sm`（轻微阴影）、`shadow-md`（浮起效果）

## 导航结构

### 页面结构
- 单页应用，通过 Tabs 切换两种模式
- 不需要 TabBar 底部导航

### 模式切换
使用 `Tabs` 组件在两种模式之间切换：
- "基础模式" - 简单任务拆解
- "干大事模式" - DDL倒排计划

## 小程序约束

### 包体积优化
- 图标使用 `lucide-react-taro`（已预安装）
- 不打包图片，所有图片走 TOS 对象存储

### 图片策略
- 头像、封面等使用 TOS 对象存储 URL
- 不使用占位符服务

### 性能优化
- 使用 `lazyLoad` 图片懒加载
- 虚拟列表（任务数量 > 50 时）

## 交互设计原则

### 反馈机制
- 点击复选框：立即显示完成动画 + Toast 反馈
- 任务拆解中：显示 Skeleton 加载状态
- 任务创建成功：自动滚动到任务列表

### 进度可视化
- 进度条实时更新，使用 `Progress` 组件
- 完成百分比用大字号显示
- 已完成步骤划线显示

### 空状态
- 无任务时显示引导提示（鼓励用户创建第一个任务）
- 拆解失败时显示错误提示 + 重试按钮

## 设计禁忌

**不要做**：
- ❌ 不要使用科技蓝 + 紫色渐变（太普通、无特色）
- ❌ 不要使用深色模式（失去健身房明亮感）
- ❌ 不要过度使用动画（保持轻量高效）
- ❌ 不要堆砌图标和装饰（保持简洁清晰）
- ❌ 不要用红、绿、蓝三原色（太刺眼）

**要坚持**：
- ✅ 主色系保持在橙色和蓝色
- ✅ 保持白色背景的明亮感
- ✅ 进度条必须有视觉冲击力
- ✅ 每个步骤都要清晰可点

## 特殊组件规范

### 进度条
- 使用 `@/components/ui/progress`
- 主任务进度：粗进度条 + 百分比文字
- 单个步骤进度：细进度条

### 复选框
- 使用 `@/components/ui/checkbox`
- 完成后显示打勾动画
- 已完成项文字添加删除线

### 时间选择
- 基础模式：不需要时间
- 干大事模式：日期选择器 + 时间输入
